# 📖 คู่มือปฐมนิเทศทีมงานไฟฟ้าและระบบสมองกลฝังตัว (Embedded & Electrical Team Handbook)
## ยินดีต้อนรับสู่ทีมพัฒนา VCU และระบบไฟฟ้ารถประหยัดพลังงาน EcoMileLab!

คู่มือฉบับย่อนี้เขียนขึ้นมาเพื่อให้สมาชิกใหม่ของทีมที่มีความรู้เริ่มต้นจากศูนย์ (0) หรือเปลี่ยนสายงานเข้ามา ได้เข้าใจโครงสร้างสถาปัตยกรรมไฟฟ้าและการเขียนโปรแกรมควบคุมรถแข่งเพื่อลุยศึก **Shell Eco-Marathon** ร่วมกันได้อย่างรวดเร็ว

## ⚡ 1. โครงสร้างฮาร์ดแวร์และสมองกล VCU (ESP32)
บอร์ดประมวลผลหลักของรถยนต์ไฟฟ้าเราเรียกว่า **VCU (Vehicle Control Unit)** ซึ่งใช้ชิปไมโครคอนโทรลเลอร์ **ESP32** (รันที่ความเร็ว 240MHz มีแรมภายใน 520KB)

### 📌 ทำไมต้องใช้ FreeRTOS (ระบบปฏิบัติการเวลาจริง)?
เราไม่ใช้คำสั่งหน่วงเวลาล็อกแบบทั่วไป (`delay()`) เนื่องจากระบบความปลอดภัยและวงจรตัดไฟฉุกเฉินต้องสแกนตรวจสอบทุกๆ มิลลิวินาที หากซอฟต์แวร์ค้าง รถจะพุ่งชนและเกิดอันตรายได้ เราจึงแบ่งการประมวลผลออกเป็นสองส่วนบน CPU 2 แกนหลัก:
* **Core 1 (คอร์ระบบควบคุมหลัก):** 
  * ทำหน้าที่สแกนปุ่ม E-Stop และ Dead-man Switch
  * อ่านค่าจากเซนเซอร์คันเร่ง (ADC) เพื่อสั่งระดับสัญญาณควบคุม PWM ไปยังอินเวอร์เตอร์ขับเคลื่อนมอเตอร์
* **Core 0 (คอร์ระบบรองและประสานงาน):**
  * ทำหน้าที่สตรีมข้อมูลวัดไฟจากเซนเซอร์ (แรงดัน, กระแส) แพ็กเป็นข้อความ JSON ยิงผ่าน LoRa ส่งกลับไปยังเต็นท์พิทสต็อป

## 🔌 2. ระบบสื่อสารและเซนเซอร์ทางไฟฟ้าที่ใช้บ่อย
บอร์ด VCU ของเราต้องดึงข้อมูลจากชิปเซนเซอร์ต่างๆ รอบคันรถ ผ่านสายสัญญาณสื่อสารดังนี้:

### A. โปรโตคอล I2C (Inter-Integrated Circuit)
* ใช้สายสัญญาณ 2 เส้นในการสื่อสาร คือ **SDA** (รับส่งข้อมูล) และ **SCL** (สัญญาณนาฬิกา) พ่วงอุปกรณ์ได้หลายตัว
* **INA226 (ไอซีวัดพลังงาน):** คอยวัดแรงดันแบตเตอรี่หลักและกระแสไฟฟ้าที่ไหลออกจากแบตเตอรี่ไปมอเตอร์
* **AS5600 (เซนเซอร์แม่เหล็ก):** ติดอยู่ที่ปลายเพลามอเตอร์เพื่อใช้อ่านมุมหมุนของแกนมอเตอร์แบบ 12-bit (ละเอียด 0 ถึง 4095 ระดับ) เพื่อคำนวณรอบความเร็ว RPM

### B. โปรโตคอล UART (Serial)
* ใช้สายสัญญาณ 2 เส้นคือ **TX** (ส่ง) และ **RX** (รับ)
* ใช้สำหรับส่งข้อมูลรายงานจาก VCU เข้าสู่บอร์ด Telemetry หรือคอมพิวเตอร์พกพาพิทเพื่อมอนิเตอร์กราฟแบบเรียลไทม์

## 🛡️ 3. ระบบความปลอดภัยไฟฟ้ากำลังสูง (High Voltage Safety)
รถแข่งของเราใช้ระบบไฟขับเคลื่อนแรงดันสูง (HV System) จึงจำเป็นต้องมีระบบนิรภัยตามกฎเกณฑ์สากล:

### A. DC Contactor & Emergency Switch (E-Stop)
* **Contactor** เปรียบเสมือนสวิตช์ไฟฟ้าที่ใช้แรงดันต่ำ 12V ไปควบคุมแม่เหล็กไฟฟ้าให้ดูดหน้าสัมผัสต่อไฟ 48V เข้าสู่ตัวรถ
* **E-Stop (ปุ่มกดแดงหัวเห็ด):** ต่ออนุกรมตัดไฟเลี้ยงขดลวดแม่เหล็กของ Contactor โดยตรงทางกายภาพ (ฮาร์ดแวร์) เมื่อกดปุ่มนี้ แม่เหล็กจะหยุดทำงาน หน้าสัมผัสจะเด้งจากกันทันทีภายใน 50ms โดยไม่ต้องผ่านซอฟต์แวร์ของสมองกล (กฎ Fail-safe)

### B. วงจรหน่วงไฟเข้าบอร์ด (Pre-charge Circuit)
* ก่อนที่จะสั่งให้ Contactor หลักประกบหน้าสัมผัส ตัวเก็บประจุภายในอินเวอร์เตอร์ขับเคลื่อนมอเตอร์จะมีสถานะเป็น 0V เสมือนขั้วบวกลบช็อตกันชั่วขณะ
* หากเราต่อไฟตรงทันที จะเกิดสปาร์ก (ประกายไฟ) ขนาดใหญ่ส่งผลให้หน้าสัมผัส Contactor เสียหายและไหม้ได้
* **ทางแก้:** เราเปิดให้กระแสไฟไหลผ่าน **ตัวต้านทานขนาดใหญ่ (Pre-charge Resistor)** ก่อนเพื่อค่อยๆ สะสมแรงดันไฟฟ้าให้กับตัวเก็บประจุอย่างปลอดภัย เมื่อชาร์จประจุได้ประมาณ 90% แล้ว VCU จึงจะสั่งประกบหน้าสัมผัสของ Contactor หลักทำงาน

## 🔄 4. ระบบควบคุมมอเตอร์แบบ FOC (Field Oriented Control)
เพื่อให้รถของเราวิ่งประหยัดพลังงานที่สุด เราจะใช้ระบบ FOC ในการขับเคลื่อนมอเตอร์แทนระบบทั่วไป:
1. VCU อ่านมุมเพลามอเตอร์ผ่านเซนเซอร์ AS5600
2. แปลงค่ากระแสไฟฟ้า 3 เฟสที่ไหลจริงให้เข้าสู่แกนสองมิติเสมือนกระแสมอเตอร์ DC ทั่วไป
3. ใช้ระบบควบคุม **PID Controller** เพื่อลดระดับกระแสพลังงานส่วนเกินที่สูญเสียไปเป็นความร้อน และดึงพลังงานไปสร้างทอร์กหรือแรงหมุนล้อได้มีประสิทธิภาพสูงสุด

## 🛠️ 5. ขั้นตอนสำหรับทีมพัฒนาที่เข้ามาปรับโค้ดใน Repo
เมื่อคุณ Clone งานชิ้นนี้ไปศึกษาและเริ่มต่อยอด:
1. **ทำการสร้างกิ่งก้านสาขาของโค้ดตนเองเสมอ:**
   ```bash
   git checkout -b feature/your-name-dev
   ```
2. **เมื่อเขียนเสร็จสิ้นและต้องการส่งงาน:**
   ```bash
   git add .
   git commit -m "อธิบายงานที่แก้ไขย่อๆ"
   git push origin feature/your-name-dev
   ```
3. ทำการเปิด **Pull Request (PR)** บนหน้าเว็บ GitHub เพื่อให้หัวหน้าทีมไฟฟ้าตรวจสอบความปลอดภัยของวงจรและโค้ด ก่อนอนุมัติรวมโค้ดเข้าสู่ระบบรถแข่งหลัก
